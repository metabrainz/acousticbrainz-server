AcousticBrainz API

Before diving into the API details, please have a look at the sample data below.

Getting data

To fetch the JSON documents for a given MusicBrainz recording ID, you can construct the following endpoints for low-level and high-level: 

   GET http://acousticbrainz.org/<MBID>/low-level
   GET http://acousticbrainz.org/<MBID>/high-level

Examples:

   GET http://acousticbrainz.org/96685213-a25c-4678-9a13-abd9ec81cf35/low-level
   GET http://acousticbrainz.org/96685213-a25c-4678-9a13-abd9ec81cf35/high-level

You can find out how many low-level documents we have for a given recording using the following endpoint: 

   GET http://acousticbrainz.org/<MBID>/count

Example:

   GET http://acousticbrainz.org/96685213-a25c-4678-9a13-abd9ec81cf35/count

If there's more than one document, you can specify a number argument n:

   GET http://acousticbrainz.org/<MBID>/low-level?n=1

Documents are counted from 0 to count-1.

Example:

   GET http://acousticbrainz.org/96685213-a25c-4678-9a13-abd9ec81cf35/low-level?n=1

Submitting data

The submit API endpoint is:

   POST http://acousticbrainz.org/<MBID>/low-level

It expects a JSON payload, containing the output of a feature extractor (see the contribute page).

Sample data

AcousticBrainz stores two levels of data: low-level and high-level.

Both types of information are JSON documents, two documents per each MusicBrainz ID for a recording.

The low-level data includes acoustic descriptors characterizing overall loudness, dynamics and spectral shape of a signal, rhythm descriptors (including beats per minute), and tonal information (including keys and scales). The low-level data is stable, compared to the higher-level data. This hopefully means that we will not have to recompute this data from audio files too frequently. More detailed information about the low-level data is provided in the official documentation for Essentia toolkit.
