Contributing to AcousticBrainz

In order to contribute acoustic data to AcousticBrainz, you will need to have a music collection that contains MusicBrainz IDs. This usually means that your collection will need to have been tagged with a MusicBrainz aware tagger like Picard or other MusicBrainz enabled taggers. If you have a collection of lossless FLAC files that would be ideal, but having lossy also helps the project. Our tools support the following file extensions: mp3 ogg oga flac mp4 m4a m4r m4b m4p aac wma asf mpc wv spx tta 3g2 aif aiff ape

Contributing Code / Documentation

If you're an open source geek and would like to tinker with the source code to AcousticBrainz, take a look at the following list of repositories that make up the project:

  • essentia: The audio analysis engine that underpins AcousticBrainz.
  • acousticbrainz-server: The server that runs the web site and manages the JSON document collection.

We have a list of ideas which describe some of the things we want to do with the AcousticBrainz database. If any of these ideas interest you, get in touch with us!

If you find a bug in AcousticBrainz, please submit report to our bug tracker.
